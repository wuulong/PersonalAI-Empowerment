# THOUGHT-LOG-MANAGER Skill (v2.3 - MT Trigger)

用於在與 AI 的高速協作過程中，精準捕捉人類的「心流、驚訝與領悟 (Meta-insights)」，並自動掛鉤當時的技術上下文。

## 觸發條件
- 當使用者訊息以 `MT:: ` (M, T, 兩個冒號加上一個空格) 開頭時觸發。`T` 代表 Trigger，用以區隔命令本身與後續生成的純資料標籤。

## 執行動作
1. **停止目前任務**: 優先處理此筆心流紀錄。
2. **初始化檔案 (Init)**: 若當天紀錄檔 (`TL_{YYMMDD}.md`) 尚未建立，首行必須寫入 `# QA`。
3. **產出摘要 (Summarization)**: 根據使用者的 `MT::` 內容，生成一個 15 字以內的短摘要。
4. **生成條目序號 (Indexing)**: 讀取當天檔案，確定目前的 Q 序號（例如當前已有 Q1, Q2，則新條目為 Q3）。
5. **上下文萃取 (Extraction)**:
    - **QS::**: 標記 [Workspace Name] | [Session ID] | [Keywords]。必須包含最近 2-3 個 Q 的完整 Prompt 內容。
    - **AS::**: 提供最近 2-3 個 A 的精簡摘要與產出路徑。
6. **心流紀錄 (Capture)**:
    - **M::**: 完整記錄使用者輸入的原始感觸（將觸發關鍵字 `MT::` 替換為標準資料標籤 `M::`）。
7. **AG 補述 (Reflection)**:
    - **AG::**: 進行技術解構。解釋 Agent 剛才的行為邏輯、使用了什麼策略，以及為什麼會產生讓使用者驚訝的效果。
8. **檔案持久化 (Persistence)**:
    - 將上述內容以追加模式寫入 `@/thought-logs/TL_{YYMMDD}.md`。
    - **格式限定**: 每次輸出最前面必須是 `## Q[數字順序] - [心流摘要]`。

## 執行規範 (Rules)
- **路徑規範**：本技能內所有以 `@/` 開頭的路徑均指代 **Workspace Root**。

## 範例格式
```markdown
# QA

## Q1 - 盲點審計協議的建立
- QS:: bmad-pa | session_abcd | [使用者之前的 Prompt 內容...]
- AS:: [最近的產出說明]
- M:: [使用者在 MT:: 之後輸入的感觸]
- AG:: [Agent 的技術補充與反思]

## Q2 - [下一個摘要]
...
```
