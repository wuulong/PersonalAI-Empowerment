# 工作日誌管理員 (Daily Work Log Manager)

協助使用者即時記錄今日成果與開發歷程。

## 體系定位：底層 - 工作日誌 (Wlog)
本技能處於「三層執行與勾稽架構」的**底層**。
- **性質**：歷程性 / 結果性。
- **功能**：紀錄每日工作的細項、進展、微調與最終產出的成績，作為中層（Task List）執行狀況的真實反映。

## 核心目標
1. **產出優先**：優先記錄具體的產出（文件、文章、資料、記錄），而非單純的技術進步或 Skill 建立。
2. **戰略對齊**：確保每日日誌與月度大目標（如：寫書、標竿專案）掛鉤，避免重大進展「失蹤」。
3. **即時性**：在重要的階段性任務完成後立即更新日誌。
4. **關聯性**：記錄 Skill 的建立是為了支撐哪項具體的產出成果。

## 檔案路徑
- **日誌存放**：`@/workmgr/work-logs/WL_YYYY-MM-DD.md`
- **月度摘要**：`@/workmgr/work-logs/WL_YYYY-MM.md` (摘要該月每一天的核心歷程與成果)

## 內容架構
1. **歷程與經驗 (Chronology & Insights)**
   - [每日] 詳細事件、Command 與檔案軌跡。
   - [每月] 每日關鍵事件摘要，需對齊月度里程碑。
2. **產出成果 (Deliverables)**
   - [每日] 具體產出清單。
   - [每月] 該月重要里程碑與累積成果。

## 執行 SOP

### Phase 1: 初始化 (Daily Init)
- 檢查今日的日誌檔案 `@/workmgr/work-logs/WL_YYYY-MM-DD.md` 是否存在。
- 若不存在，則根據 `templates/work_log_template.md` 建立。

### Phase 2: 階段性增量更新 (Continuous Update)
- 每當完成一個子任務或遇到重要卡點解決後，主動執行更新。
- **歷程記錄**：包含當時的心境轉折或技術決策。
- **Command Trace**：從工具調用清單中提取關鍵指令。

### Phase 3: 總結成果 (Result Finalization)
- 在對話接近尾聲或當天工作結束前，整理「今日成果」清單。

### Phase 4: 三層結構勾稽校準 (Three-Tier Alignment Audit)
- **向上勾稽 (To Task List)**：檢查今日 Wlog 中完成的事項，是否對應 `@/workmgr/tasks/TASKS.md` 中的 Task ID。若有新產生的 Task 應建議補回清單。
- **向下紮根 (Fact Check)**：
  1. 「里程碑」中提到的重大進展，是否在「每日歷程」中有對應紀錄？
  2. 是否有「為了蓋房子（專案）而磨錘子（技術）」卻只記了磨錘子的情況？
  3. 若發現漏失，應主動回填並向使用者確認。

## 注意事項
- **防止「技術噪音」淹沒成果**：避免日誌變成單純的 git log 或 command history。
- **不要等待請求**：Agent 應在感知到「進展」時主動執行更新日誌的動作。
- **詳盡性（累加而不摘要）**：歷程部分越細越好。應採取 **累加 (Append)** 而非摘要 (Summarize) 的方式記錄今日事件。即便內容變長，也能透過 Git 歷史找回，保持原汁原味的開發軌跡是為了應對未來「遺忘」的保險。
